# AI个人健康信息管理助手架构设计文档 (V1.0)

## 1. 项目概述

### 1.1 项目背景
在信息过载的医疗健康领域，个人用户面临专业信息检索困难、健康数据碎片化、医学科普内容质量参差不齐的痛点。本应用旨在利用人工智能技术，构建一个可信、易用且智能的个人健康信息管理中枢，为用户提供从症状初判到健康档案管理的全流程辅助。

### 1.2 产品定位
一款以权威医疗信息为基石，以AI为驱动核心的C端个人健康信息管理助手。应用严格定位于 **信息提供、整理与科普**，不提供任何诊断结论或诊疗方案建议，所有输出均需标明“仅供参考，不能替代专业医疗诊断”。

### 1.3 核心价值主张
*   **信息降噪与权威直达**：聚合并智能解析权威医学指南、顶级医院科室信息及优质科普内容，为用户提供清晰、可靠的信息参考。
*   **个人健康数据枢纽**：结构化存储和管理用户的健康交互记录与病历资料，形成持续更新的个人健康档案。
*   **个性化知识伴侣**：根据用户的查询历史和健康档案，主动提供匹配的、通俗易懂的医学科普知识。

### 2.2 核心服务模块分解

#### 智能查询与解析服务
*   **职责**：处理用户输入的自然语言病症描述，生成专业、结构化的查询请求，并调用AI模型和知识库获得综合答复。
*   **核心流程**：
    1.  **意图识别与实体抽取**：识别用户意图（如“找医院”、“查病因”、“读指南”）并抽取关键医学实体（如症状“持续性头痛”、疾病“高血压”）。
    2.  **查询规划与路由**：根据意图，并行或串行触发不同查询（如同时查询疾病知识和相关医院排名）。
    3.  **检索增强生成 (RAG)**：将查询向量化，从 **向量知识库** 中检索最相关的指南片段、科普文章摘要、医院科室信息。将“用户问题+检索到的权威片段”组合成提示词，提交给 **大语言模型** 生成整合性、口语化的答复。
    4.  **结构化输出**：输出包含“核心解读”、“可能相关疾病方向”、“建议就诊科室”、“权威参考来源（明确引用来源名称、链接）”及“相关顶级医院推荐（来自排行榜）”的结构化结果。

#### 健康档案管理服务
*   **职责**：管理用户所有的健康数据。
*   **核心数据**：
    *   **交互历史**：每次智能查询的“用户问题”和“AI答复”，结构化存储。
    *   **个人病历库**：用户上传的检查报告、处方笺图片/PDF，通过OCR（光学字符识别）和NLP技术提取关键信息（检查项目、结果、诊断、药品），形成结构化数据。
    *   **健康时间线**：将交互历史、结构化病历数据、用户手动记录的健康指标（如血压）按时间轴自动整合，形成可视化健康档案。

#### 内容管理与推荐服务
*   **职责**：管理权威数据源，并实现个性化科普推送。
*   **数据源管理**：
    *   **内部知识库**：定期爬取或接入合作的权威医学指南、丁香通等专业网站的科普文章，经清洗、向量化后存入 **向量知识库**。
    *   **外部榜单API**：接入或定期更新复旦版《中国医院排行榜》及卫健委官方评价数据，作为医院推荐的依据。
    *   **推荐引擎**：基于用户的健康档案标签、查询历史，从科普库中筛选匹配的文章进行主动推送。

### 2.3 核心数据结构设计
*   **向量知识库**：存储权威医学文本（指南、科普）的向量化嵌入，用于相似性检索。每个向量关联原文的元数据（标题、来源、网址、发布日期）。
*   **业务数据库（主要表）**：
    *   **用户表**：用户基础信息。
    *   **交互记录表**：存储每次QA对，关联 **疾病实体标签**。
    *   **病历文档表**：存储用户上传的原始文件及OCR解析后的结构化JSON数据。
    *   **医院科室榜单表**：存储从外部数据源同步的医院、科室排名及核心评价信息。

## 3. 关键技术选型与实现路径

### 3.1 推荐技术栈 (MVP最小可行产品阶段)
| 组件 | 推荐技术/服务 | 说明 |
| :--- | :--- | :--- |
| **前端** | React Native / 微信小程序 | 跨平台，开发效率高，生态丰富。 |
| **后端框架** | Python (FastAPI) / Node.js | 快速构建RESTful API，异步支持好，适合AI集成。 |

| **向量数据库** | Pinecone / Weaviate / 腾讯云VectorDB | 托管服务，简化知识库的构建与检索。 |

| **对象存储** | 阿里云OSS / 腾讯云COS | 存储用户上传的病历图片、PDF等原始文件。 |
| **运维部署** | Docker, Kubernetes (K8s) | 容器化部署，保证服务弹性和可扩展性。 |

### 3.2 核心AI实现：检索增强生成 (RAG) 流程
这是保证信息 **权威性** 和 **可控性** 的核心。

```python
# 简化的RAG流程伪代码
def intelligent_health_query(user_query, user_history):
    # 1. 意图识别与实体抽取 (可基于微调的NER模型或Prompt工程)
    intent, medical_entities = medical_nlp_model.analyze(user_query)

    # 2. 根据意图，从向量知识库检索相关权威文档片段
    if intent == "disease_inquiry":
        relevant_chunks = vector_db.search(query=medical_entities, filter={"source_type": "guidebook"})
    elif intent == "hospital_recommendation":
        relevant_chunks = vector_db.search(query=medical_entities, filter={"source_type": "hospital_ranking"})

    # 3. 构建提示词 (Prompt)，注入权威参考信息
    prompt = f"""
    你是一个专业的医疗信息助手。请**严格基于**以下提供的权威信息，用通俗易懂的语言回答用户问题。
    如果提供的信息不足以回答，请如实告知，不要编造。

    【权威参考信息】:
    {relevant_chunks}

    【用户历史健康上下文】:
    {user_history}

    【用户当前问题】:
    {user_query}

    【你的回答格式要求】:
    1. 核心解读：概括性回答。
    2. 可能方向：列出信息中提及的相关疾病或科室。
    3. 行动建议：如“建议挂【XX科】门诊进一步检查”。
    4. 参考来源：明确列出本回答依据的来源（如《XXX指南》、复旦大学医院管理研究所《2023年度中国医院排行榜》）。
    5. 免责声明：结尾必须加上“*内容仅供参考，不能替代专业医师诊断和治疗*”。
    """
    # 4. 调用大语言模型生成最终答复
    final_answer = llm_api.call(prompt=prompt)
    return final_answer, relevant_chunks  # 保存结果和溯源信息
```

## 4. 路线图与迭代计划

### 第一阶段：MVP (1-3个月)
*   **目标**：验证核心功能闭环。
*   **交付物**：实现基于关键词和简单向量的症状-科室匹配、手动上传病历（图片）、展示基础版复旦医院排行榜。
*   **技术验证重点**：RAG流程在医疗垂直领域的准确性测试，核心数据源（如一个科室的指南）的整合效果。

### 第二阶段：功能深化 (4-6个月)
*   **目标**：提升智能化与用户体验。
*   **交付物**：部署完整的 **向量知识库** 支持自然语言查询，实现病历OCR与结构化，推出个性化科普推送。
*   **技术建设重点**：构建完整的医学知识向量化流水线，优化推荐算法。

### 第三阶段：生态扩展 (7-12个月)
*   **目标**：从工具扩展到平台。
*   **交付物**：支持用户授权数据用于随访研究（合规前提下），探索与互联网医院、保险机构的API级合作。
*   **核心任务**：建立严格的数据隐私与安全合规体系，探索可持续的商业模式。

## 5. 核心风险与应对策略

### 医学信息准确性风险
*   **策略**：建立严格的信息源白名单机制和更新审核流程。所有AI生成回答必须附带可追溯的权威来源引用。引入医学专家顾问团进行内容审核。

### 数据安全与隐私风险
*   **策略**：健康数据全链路加密存储与传输（符合等保三级/GDPR要求）。提供清晰的数据授权管理选项，用户数据完全去标识化后用于模型训练。

### 法规与合规风险
*   **策略**：应用不上线任何诊断功能，所有页面显著位置标注免责声明。积极申请医疗器械软件（如符合，申请一类或二类）备案，确保业务合法合规。

### 技术实施风险
*   **策略**：采用 **检索增强生成(RAG)** 而非模型微调作为初期核心方案，以控制模型“幻觉”并降低训练成本。优先使用成熟的云服务和托管平台，降低工程复杂度。
